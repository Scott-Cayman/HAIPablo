# 钉钉权限验证功能文档

## 一、功能概述

本系统集成了钉钉 OAuth 2.0 登录认证和钉钉通讯录同步功能，实现：

1. **钉钉 SSO 单点登录** - 用户可通过钉钉扫码授权直接登录系统
2. **邮箱域名限制** - 仅允许 `@himice.com` 域名的企业邮箱用户登录
3. **自动用户创建** - 首次登录自动创建系统账号
4. **部门自动同步** - 登录时自动关联用户在钉钉中的部门信息
5. **全量通讯录同步** - 管理员可从钉钉拉取完整的部门架构和用户数据

## 三、API 接口详解

### 3.1 获取钉钉登录地址

**接口**: `GET /api/auth/dingtalk/login-url`

**功能**: 生成钉钉 OAuth 授权 URL，引导用户跳转进行扫码授权。

**请求参数**: 无

**响应示例**:
```json
{
  "url": "https://login.dingtalk.com/oauth2/auth?client_id=xxx&redirect_uri=xxx&...",
  "state": "abc123random"
}
```

**核心逻辑**:

```typescript
export async function GET(request: NextRequest) {
  const clientId = process.env.DINGTALK_CLIENT_ID;
  const redirectUri = process.env.DINGTALK_REDIRECT_URI;
  const state = Math.random().toString(36).substring(2, 15);
  
  // 构建钉钉授权 URL
  const loginUrl = new URL('https://login.dingtalk.com/oauth2/auth');
  loginUrl.searchParams.append('client_id', clientId);
  loginUrl.searchParams.append('redirect_uri', redirectUri);
  loginUrl.searchParams.append('response_type', 'code');
  loginUrl.searchParams.append('scope', 'openid corpid');  // openid: 获取用户身份, corpid: 获取企业 ID
  loginUrl.searchParams.append('state', state);
  loginUrl.searchParams.append('prompt', 'consent');  // 每次都显示授权确认页

  return NextResponse.json({ url: loginUrl.toString(), state });
}
```

**参数说明**:

| 参数 | 说明 |
|------|------|
| `client_id` | 钉钉应用的 Client ID |
| `redirect_uri` | 授权成功后的回调地址 |
| `response_type` | 固定为 `code`，返回授权码 |
| `scope` | `openid corpid` - 获取用户 OpenID 和企业 ID |
| `state` | 随机字符串，防止 CSRF 攻击 |
| `prompt` | `consent` - 每次授权都要求用户确认 |

### 3.2 钉钉 OAuth 回调处理

**接口**: `POST /api/auth/dingtalk/callback`

**功能**: 接收钉钉授权码，交换用户信息，在本地系统中完成登录或注册。

**请求参数**:
```json
{
  "authCode": "授权码",
  "state": "状态参数"
}
```

**完整业务流程**:

```typescript
export async function POST(request: NextRequest) {
  // 1. 接收授权码
  const { authCode } = await request.json();
  const { clientId, clientSecret } = process.env;

  // 2. 用授权码换取用户 Access Token
  const tokenResponse = await fetch('https://api.dingtalk.com/v1.0/oauth2/userAccessToken', {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify({
      clientId,
      clientSecret,
      code: authCode,
      grantType: 'authorization_code',
    }),
  });
  const tokenData = await tokenResponse.json();
  const userAccessToken = tokenData.accessToken;

  // 3. 获取用户基础信息 (v1.0 API)
  const userResponse = await fetch('https://api.dingtalk.com/v1.0/contact/users/me', {
    method: 'GET',
    headers: {
      'x-acs-dingtalk-access-token': userAccessToken,
    },
  });
  const userData = await userResponse.json();
  // userData 包含: unionId, openId, nick, avatarUrl, mobile, email, orgEmail, deptIdList

  // 4. 补充完整用户信息 (通过企业级 API)
  // 由于 v1.0 API 可能不返回 orgEmail，需要调用 oapi.dingtalk.com 的 v2/user/get 接口
  // 4.1 获取企业 App Token
  const appTokenRes = await fetch(`https://oapi.dingtalk.com/gettoken?...`);
  const appAccessToken = appTokenRes.access_token;
  
  // 4.2 通过 unionid 获取 userid
  const uidRes = await fetch(`https://oapi.dingtalk.com/topapi/user/getbyunionid?...`, {
    body: JSON.stringify({ unionid: unionId })
  });
  
  // 4.3 通过 userid 获取完整信息（含 org_email）
  const detailRes = await fetch(`https://oapi.dingtalk.com/topapi/v2/user/get?...`, {
    body: JSON.stringify({ userid: userid })
  });

  // 5. 邮箱域名校验 - 必须是 @himice.com
  const userEmail = orgEmail || email;
  if (!userEmail?.endsWith('@himice.com')) {
    return NextResponse.json({ error: '仅限企业邮箱用户登录' }, { status: 403 });
  }

  // 6. 同步部门信息
  if (deptIdList?.length > 0) {
    const primaryDeptId = String(deptIdList[0]);
    // 6.1 获取部门名称
    const deptResponse = await fetch(`https://api.dingtalk.com/v1.0/contact/departments/${primaryDeptId}`, {...});
    const deptName = deptResponse.name || `钉钉部门_${primaryDeptId}`;
    
    // 6.2 在数据库中创建或更新部门
    await prisma.department.upsert({...});
  }

  // 7. 查找或创建本地用户
  let user = await prisma.user.findFirst({
    where: {
      OR: [
        { dingtalkUnionId: unionId },
        { email: userEmail },
      ],
    },
  });

  if (!user) {
    // 首次登录，自动创建账号
    user = await prisma.user.create({
      data: {
        username: userEmail,
        email: userEmail,
        password: Buffer.from('12345678').toString('base64'), // 默认密码
        name: nick,
        avatar: avatarUrl,
        dingtalkUnionId: unionId,
        dingtalkOpenId: openId,
        departmentId,
        lastLoginAt: new Date(),
      },
    });
  } else {
    // 更新钉钉关联信息和最后登录时间
    user = await prisma.user.update({
      where: { id: user.id },
      data: {
        dingtalkUnionId: unionId,
        dingtalkOpenId: openId,
        avatar: avatarUrl || user.avatar,
        departmentId: departmentId || user.departmentId,
        lastLoginAt: new Date(),
      },
    });
  }

  // 8. 返回用户信息给前端
  return NextResponse.json({
    id: user.id,
    username: user.username,
    email: user.email,
    name: user.name,
    avatar: user.avatar,
    role: user.role,
    credits: user.credits,
  });
}
```

### 3.3 钉钉通讯录同步

**接口**: `POST /api/dingtalk/sync`

**功能**: 管理员手动触发，从钉钉拉取完整的部门架构和人员信息，同步到本地数据库。

### 5.4 错误信息脱敏

API 错误不返回内部实现细节：

## 八、常见问题

### Q1: 为什么需要同时调用 v1.0 和 oapi 两个 API？

**原因**: 
- `api.dingtalk.com/v1.0` 是新版 API，接口规范但可能不返回 `org_email`
- `oapi.dingtalk.com` 是旧版 API，需要 `access_token` 参数，但返回信息更完整

**解决方案**: 先用 v1.0 获取基础信息，再用 oapi/v2/user/get 补充完整信息。

### Q2: 同步用户时，钉钉用户没有 org_email 怎么办？

**情况分析**: 
1. 用户未在钉钉绑定企业邮箱
2. 应用没有申请"企业邮箱"权限
3. 用户所在部门限制了邮箱可见性

**处理方式**: 系统使用 `@dingtalk.local` 作为占位邮箱，允许该用户登录但需要后续手动完善信息。

### Q3: 如何排查同步失败问题？

**排查步骤**:

1. 检查控制台日志中的 `errcode` 和 `errmsg`
2. 确认钉钉应用已开通"通讯录"相关权限
3. 验证 `DINGTALK_CLIENT_ID` 和 `DINGTALK_CLIENT_SECRET` 是否正确
4. 检查企业 Access Token 是否过期
5. 确认网络能够访问钉钉 API

### Q4: 如何添加新的邮箱域名白名单？

修改 `app/api/auth/dingtalk/callback/route.ts` 中的域名校验逻辑：

```typescript
const ALLOWED_DOMAINS = ['@himice.com', '@example.com']; // 添加更多域名

const userEmail = orgEmail || email;
const isAllowed = ALLOWED_DOMAINS.some(domain => userEmail?.endsWith(domain));

if (!isAllowed) {
  return NextResponse.json({ 
    error: `登录失败：仅限 ${ALLOWED_DOMAINS.join('、')} 域名用户登录` 
  }, { status: 403 });
}
```
